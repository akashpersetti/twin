## Project: Wingman (Self-Evaluating Agentic Co-Worker)
Wingman is a self-evaluating agentic co-worker built with LangGraph, FastAPI, Lambda, and DynamoDB, live at wingman.akashpersetti.com. Akash designed a worker-evaluator state machine in LangGraph where a structured-output evaluator checks worker responses against user-defined criteria and reinjects feedback for up to 5 retry turns, raising answer-validity rates before surfacing a result. He made the agent serverless-safe by reconstructing LangGraph state from DynamoDB on every request instead of an in-memory checkpointer, eliminating cold-start state loss across Lambda invocations. He integrated 5 tool domains through LangChain ToolNode and bind tools: web search (Serper), Wikipedia, sandboxed Python REPL, sandboxed file I/O, and Pushover notifications. He shipped the full stack on AWS with zero long-lived credentials: containerized FastAPI with Mangum, pushed to ECR, and provisioned Lambda, API Gateway v2, DynamoDB, and CloudFront through a Terraform-managed CI/CD pipeline using GitHub Actions OIDC.

## Project: Twin (Streaming AI Digital Twin)
Twin is a streaming AI digital twin built with FastAPI, Next.js, AWS Bedrock (Claude Sonnet), S3, and CloudFront, live at akashpersetti.com. Akash built a streaming personal-website agent grounded in a persona assembled at Lambda cold-start from a LinkedIn PDF, career summary, and style guide (parsed via pypdf) and injected into a Bedrock Claude Sonnet system prompt. He hardened it against hallucination and prompt injection with a strict system prompt that forbids fabricated facts, blocks common jailbreak patterns, and constrains every response to verified persona context. He cut full-stack deployment to a single command across dev, test, and prod by wiring GitHub Actions and AWS OIDC to chain Docker build, Lambda upload, Terraform apply, Next.js export, S3 sync, and CloudFront invalidation. He persisted per-user conversation history at near-zero cost using session-keyed JSON in a private S3 bucket, with session IDs generated client-side through API Gateway. The site has served around 90 unique sessions.

## Project: TallyMark (Voice + Text Agentic Expense Splitter)
TallyMark is a voice and text agentic expense splitter built with React Native (Expo), FastAPI, NeonDB (PostgreSQL), and LangGraph. Akash engineered a dual-modal LangGraph agent that parses natural-language expense descriptions from voice or text, answers balance and group queries, and executes write actions (add expense, settle debt) against live user data via tool-bound FastAPI endpoints. He implemented a graph-based debt simplification algorithm that collapses N-way IOUs into the minimum-transaction settlement, toggled per group so users see either gross splits or net balances on demand, surfaced through a 4-tab React Native frontend over a modular FastAPI and NeonDB backend. TallyMark was formerly named SplitMirror.

## Project: mcp-second-opinion (Open-Source MCP Server)
mcp-second-opinion is an open-source MCP server published to PyPI at pypi.org/project/mcp-second-opinion. Akash built and published this MIT-licensed MCP server that lets any MCP-aware agent consult rival LLMs (OpenAI, Gemini, Anthropic, Grok) mid-conversation, exposing two tools: one to query a single model and one to fan out to all enabled providers in parallel and compare answers. He unified four provider APIs behind a single LiteLLM interface, returning per-call latency, token counts, and cost; it gracefully disables providers with no API key instead of failing, and ships as a pip-installable CLI registered through standard MCP client config. The two tools are named ask_other_model and ask_the_panel.
